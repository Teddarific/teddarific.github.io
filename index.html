Tiny Search Engine
Software Design and Implementation
I implemented a Tiny Search Engine in C for one of my classes Software Design and Implementation. The Tiny Search Engine consisted of three components: crawler, indexer, and querier. The crawler visits a root webpage, and from there, explores further linked websites until a certain depth, and stores the HTML of these pages in a directory. The indexer then went through the directory created by the crawler, and created an index of all the words. The querier then accepted user input, and searched the index for documents that contain the query. The querier was also able to handle "and" and "or" statements, and ranked the results accordingly.

If you would like to see the code, please contact me at teddy.ni.19@dartmouth.edu!

Partilab
Personal Project
As a way to learn the MERN stack, I decided to puruse a project that would essentially allow people to collaborate on music playlists in real time. Partilab, at concept, allows users to create and join parties, and when part of a party, can add, upvote, and downvote songs to modify the current playlist. While still a work in progress, the web app is built on a Node.js/Express server, and utilizes Socket.io for real-time communication, YouTube's API for music, Passport.js for OAuth 2.0, Mongoose for storing parties, just to name a few technologies being used.
